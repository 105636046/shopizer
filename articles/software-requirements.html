Software requirements


Shopizer is a responsive web application built with Java. You might want to just install the application and run it or customize the application to meet your needs for building a custom online store. The following section describes what is required to run the application and optionnaly how to customize the application.

- Java version 8

- A JEE servlet container or any JEE application server (see the instructions for Apache Tomcat)

- Maven for building source files

- Oracle MySQL database (H2 database is included by default)

Java version


Check if java is installed, for this open a shell or batch window


Check which version of java is installed 

$ java -version

Check if a compiler is installed 
$ javac


If both commands responded you have a valid Java SDK installed, otherwise you can get a working copy from

Version Link URL

Java 8 http://www.oracle.com/technetwork/java/javase/downloads/index.html


Apache Maven installation


Download Maven from this url http://maven.apache.org/download.cgi
Folow the installation instructions on the Apache Maven download page


Application server / servlet container


Shopizer was tested on those servlet containers and application servers

Apache Tomcat - http://tomcat.apache.org
Jetty - http://jetty.codehaus.org/jetty/
Oracle Weblogic - http://www.oracle.com/technetwork/middleware/weblogic/overview/index.html
IBM Websphere - http://www14.software.ibm.com/webapp/download/brand.jsp?b=Websphere


Oracle MySQL


Shopizer contains by default a fully working version of HSQL DB for testing the application initialy. Although all functionality perfectly work with that database it is recommended to install and work with MySQL when going live. Some more details are available on the configuration of Shopizer to use MySQL.
