References objects


References are objects used for building main Shopizer business objects. Reference objects map to database tables which for most of them can be populated from the administration tool.

Zone


Zones represent provinces and states. Zones are loaded dynamically on the initial system startup. Zones file can be found from a static json configuration file:

shopizer/sm-core/src/main/resources/reference/zoneconfig.json

And can be reladed for each new startup. Zones exist in english and french, if you want to modify the list feel free to do so and you are welcome to add a new language to that list. Each zone refers to a valid country code that must exist before starting the insertion of new zones.

Country


Country are loaded dynamically during the initial startup. An initial static list of ISO country code is provided in com.salesmanager.core.constants.SchemaConstant class. 

Locale locale = new Locale("US");
String name = locale.getDisplayCountry(new Locale("en"));
Country list will be loaded on the first startup. For each country a zone exist in english and french, if you want to modify the list feel free to do so and you are welcome to add a new language to that list. Each zone refers to a valid country code that must exist before starting the insertion of new zones.

Language


Languages are defined in the database in the table language. Shopizer uses java.util.Locale in order to get the name of the Language.
